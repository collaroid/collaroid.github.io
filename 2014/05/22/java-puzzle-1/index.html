【Java Puzzle】取余运算和与运算的关系

在看Java Puzzle，第一个是%取余数的正负号问题。这个问题本没啥好说的，记着提到的取余运算的定义就可以：A%B=A-(A/B)*B
这样可以看出最后结果是受正负数影响的。亲测如下：

在这个问题后面，提出了一个用与运算来代替n%2的写法，即n&1。这让我想起了一段曾经的往事。。若干年前，一个月黑风高的下午，去面试，当我写出一个三目的运算来简化基偶性判断时，面试官漏出猥琐的笑容并问道，是否还能继续优化，这。。。以上为引子！

言归正传，对于n%2的操作意义，我们都知道是来取n被2除后的余数。那么，n减去所求的结果，这个数字必然是可以被2整除的。以n=9为例，以二进制的眼光来看，二进制9为1001，能被2整除的是从右向左第二位开始，均能够被2整除，所以余数必然是第一位的1。这样n%2的计算，就可以转化为n&1，来拿到最低位了。

再看一个例子会更清楚点。79%4的计算，79=64+8+4+2+1（二进制1001111），能被4整除的为从右向左第三位（4）开始均可以。那么余数必然是剩下的两位，即是11，为3。那么直接对n进行最后2位的与运算就可以拿到取余数的结果，那么79%4=79&3。

综上，同理即可推出，对于一个数取2的幂次余，即可以转化为求这个数与该2的幂减一的数进行与运算。例：n%2=n&1,n%4=n&3,n%8=n&7…
